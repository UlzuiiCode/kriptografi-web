About Caesar Cipher & Mod 256

What is Caesar Cipher?

The Caesar Cipher is one of the oldest and simplest encryption techniques. Named after Julius Caesar, who used it to protect military messages, it works by shifting each character in the message by a fixed number of positions in the alphabet (or in our case, the ASCII table).

How Does This Implementation Work?

This tool uses the full ASCII table (0-255) instead of just letters. Each character is converted to its ASCII value, then the key is added (for encryption) or subtracted (for decryption), and finally the modulo 256 operation is applied to keep the result within valid ASCII range.

Encryption Formula:

Encrypted ASCII = (Original ASCII + Key) mod 256

Decryption Formula:

Decrypted ASCII = (Encrypted ASCII - Key + 256) mod 256

What is Mod 256?

The modulo operation (mod) returns the remainder after division. "Mod 256" ensures that the result always stays within the range 0-255, which corresponds to the extended ASCII character set. This creates a "wrap-around" effect: if adding the key would exceed 255, it wraps back to 0.

Example: If a character has ASCII value 250 and the key is 10, the result would be (250 + 10) mod 256 = 260 mod 256 = 4. The value wraps around to 4.

Security Note:

While Caesar Cipher is great for learning cryptography concepts, it's not secure for real-world use. With only 256 possible keys, it can be easily broken by trying all possibilities (brute force attack). Modern encryption uses much more sophisticated algorithms.
